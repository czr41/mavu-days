# Architecture Decision Records (ADR)

Each ADR is one markdown file named `NNN-short-title.md` with:

1. Context — what forced a choice  
2. Decision — what we picked  
3. Consequences — trade-offs and follow-ups  

Start at `docs/adr/`. Examples to add soon:

- `001-auth-fastify-jwt.md`
- `002-jobs-bullmq-redis.md`
- `003-payments-mock-vs-provider.md`
